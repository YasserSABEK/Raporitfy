<action>
Replace `app/(tabs)/actions/index.tsx` with a full dashboard:

**Header:** "Suivi des actions" as screen title

**Filter bar (horizontal ScrollView at top):**
1. **Project filter chips:** "Tous" + one chip per project with open actions. Active chip = filled, inactive = outline. Uses `useAllOpenActions` data to extract unique project names.
2. **Status segment:** "Ouvertes" (default) | "En cours" | "Toutes" — horizontal button group below chips

**Action list (FlatList):**
- Uses `useAllOpenActions()` hook
- Filter by selected project + status
- Sort by deadline ascending (most urgent first)
- Pull-to-refresh

**Action card layout:**
```
┌─────────────────────────────────────┐
│ [priority color bar left 4px]       │
│ Projet : {project_name}  (small)    │
│ {description}              (2 lines)│
│                                     │
│ 👤 {owner}          📅 {deadline}   │
│ [status badge]     [retard badge?]  │
└─────────────────────────────────────┘
```

**Priority color left border:**
- basse: `#6B7280` (gris)
- moyenne: `#F59E0B` (amber)
- haute: `#EF4444` (rouge)

**Deadline color coding:**
- `> 3 days away` → green text `#10B981`
- `≤ 3 days` → amber text `#F59E0B`
- `past` → red text `#EF4444` + badge "En retard · J+{N}"

**Status badges:**
- ouverte: outline badge, grey
- en_cours: filled badge, blue `#3B82F6`
- fermee: filled badge, green `#10B981`
- reportee: filled badge, amber `#F59E0B`

**Tap on card:** Navigate to the action's visit detail screen:
```typescript
router.push(`/(tabs)/projects/${action.project_id}/visits/${action.visit_id}?t=${Date.now()}`)
```

**Long-press on card:** Alert with:
- "Ouverte" / "En cours" / "Fermée" / "Reportée" — quick status change via `useUpdateAction`
- "Annuler"

**Empty state (when no open actions at all):**
Keep existing empty state with clipboard icon: "Aucune action en cours — Les actions créées lors de vos visites de chantier apparaîtront ici."

**Empty state (when filters produce no results):**
"Aucune action ne correspond aux filtres sélectionnés."

**Count badge on tab icon:**
In `app/(tabs)/_layout.tsx`, add badge count to the Actions tab using `tabBarBadge`:
```typescript
<Tabs.Screen
  name="actions"
  options={{
    tabBarBadge: openActionsCount > 0 ? openActionsCount : undefined,
    // ... existing options
  }}
/>
```
This requires fetching the count from a provider or passing it via context. Simplest approach: use the `useAllOpenActions` hook in the layout and pass the count.

**Important performance note:** `useAllOpenActions` does a join across action_items → visits → projects. For MVP this is fine. If performance degrades, add a Supabase view later.
</action>
